## Issues encountered & resolution

- **Symptom:** `curl.exe -sI https://jameslanzon.com` shows 22
  response headers, none of them `strict-transport-security`.
  AC-6 as originally written would fail the production smoke.
  **Diagnosis:** GitHub Pages does not emit HSTS on custom domains;
  it only emits the header on `*.github.io` hosts where GitHub
  controls the parent zone and can preload it. This is a long-standing
  platform behaviour, not a configuration error — confirmed by
  comparing to other public GitHub-Pages-hosted custom domains and
  by reading the Pages public docs.
  **Fix:** Option A — AC-6 relaxed to "HSTS is OPTIONAL"; spec body,
  ADR-002, and the e2e test all updated. The 301 redirect from
  `http://` (AC-2) remains the day-zero protection. Reintroducing
  real HSTS will require a CDN in front of Pages (a new ADR
  superseding ADR-002); deferred to a future iteration.
- **Symptom:** `curl.exe -vI https://jameslanzon.com` doesn't print
  `subject:`/`issuer:` lines.
  **Diagnosis:** Windows curl uses Schannel for TLS, not OpenSSL;
  Schannel doesn't expose cert fields to curl's verbose output.
  **Fix:** Used `[System.Net.HttpWebRequest]::Create` +
  `$req.ServicePoint.Certificate` instead — same data, different
  surface. Cert dump captured in §Evidence.
